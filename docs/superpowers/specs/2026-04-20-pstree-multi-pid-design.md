# pstree 多 PID 支持

## 目标

扩展现有 `pstree` 脚本，支持指定多个 PID，合并展示共享祖先的进程树，无共同祖先时回退为独立树输出。

## 命令行接口

```
pstree <PID> [PID2] [PID3] ...
```

- 所有位置参数为 PID，至少需要一个
- 全部 PID 不存在时报错退出；部分不存在时警告但继续

## 核心算法

### 数据收集

一次 `ps -eo pid=,ppid=,args=` 获取全部进程，存入关联数组：
- `PPID[pid]` → 父进程号
- `CMD[pid]` → 命令行

### 构建子进程索引

遍历一次，建立 `CHILDREN[ppid]` → "child1 child2 ..." 数组。

### 合并逻辑

1. 对每个目标 PID，向上追溯至 PID 1，得到祖先路径
2. 多个祖先路径取最长公共前缀，找到共同祖先
3. 按共同祖先分组：
   - 同组 PID → 合并为一棵树，共享祖先链
   - 不同组 → 各组独立输出，空行分隔

### 裁剪无关分支

从共同祖先向下遍历，只展开包含目标 PID 或其祖先的分支，跳过无关子树。

### 高亮

所有目标 PID 均用红色高亮（`\033[1;31m`）。

## 输出格式

单 PID 时与当前行为一致。

多 PID 共享祖先时：

```
1 • 0 • /sbin/init
    └── 50 • 1 • /usr/bin/launcher
            └── 100 • 50 • /usr/bin/app
                    ├── 200 • 100 • /usr/bin/worker      ← 红色高亮
                    │       └── 201 • 200 • /usr/bin/task
                    └── 300 • 100 • /usr/bin/worker2     ← 红色高亮
                            └── 301 • 300 • /usr/bin/task2
```

多 PID 无共同祖先时，各组独立输出，空行分隔。

## 实现：方案 A（进程池 + 标记法）

一次性获取系统全部进程数据，在内存中构建完整的父子关系图。优势：
- 只调用一次 `ps`，性能最优
- 合并逻辑在内存中操作，清晰高效
- 多 PID 场景下不会因重复调用 `ps` 而放大性能问题

## 兼容性

保持对 macOS 和 Linux 的兼容（当前脚本已有的 `uname` 检测逻辑）。
