# pstree 多 PID 支持

## 目标

重写现有 `pstree` 脚本，支持指定多个 PID，合并展示共享祖先的进程树，无共同祖先时回退为独立树输出。使用进程池方案统一处理单 PID 和多 PID 场景。

## 命令行接口

```
pstree <PID> [PID2] [PID3] ...
```

- 所有位置参数为 PID，至少需要一个
- 重复 PID 自动去重
- 全部 PID 不存在时报错退出（exit 1），输出 `pstree: 进程号 <PID> 不存在`
- 部分 PID 不存在时输出警告 `pstree: 进程号 <PID> 不存在，已跳过`，继续处理剩余 PID
- 不支持 PID 0（内核调度器），传入时报错

## 核心算法

### 数据收集

一次 `ps -eo pid=,ppid=,args=` 获取全部进程，存入关联数组：
- `PPID[pid]` → 父进程号
- `CMD[pid]` → 命令行

### 构建子进程索引

遍历一次，建立 `CHILDREN[ppid]` → "child1 child2 ..." 数组。

### 祖先追溯（含环检测）

对每个目标 PID，沿 PPID 向上追溯至 PID 1，得到有序祖先路径 `[1, ..., target]`。追溯时使用 visited 集合检测环，遇到重复 PID 或 PPID 指向不存在的进程时终止。

### LCA 分组

1. 对每个目标 PID 得到祖先路径
2. 找到所有路径的**最低公共祖先（LCA）**：所有祖先路径中都出现的最深（离根最远）的 PID
3. 若 LCA 为 PID 1（即 PID 1 是唯一共同祖先），视为无共同祖先，回退为独立树
4. 按 LCA 分组：同组 PID 合并为一棵树，不同组独立输出，空行分隔

### 裁剪无关分支

构建**标记集合**：所有目标 PID 加上每个目标 PID 从 LCA 到自身的路径上的所有 PID。渲染树时：
- 从 PID 1 开始，始终显示到 LCA 的完整祖先链（与单 PID 行为一致）
- 从 LCA 向下，只渲染标记集合中的节点
- 标记集合中节点的兄弟节点不显示（除非兄弟本身在标记集合中）

### 高亮

所有目标 PID（命令行参数指定的）均用红色高亮（`\033[1;31m`）。

### 树根

输出始终从 PID 1 开始，展示完整的祖先链到 LCA，再展开 LCA 的子树。与现有单 PID 行为一致。

## 输出格式

每个节点格式：`PID • PPID • 命令行`（与现有格式一致）。

单 PID 时与当前行为一致。

多 PID 共享祖先时（LCA = PID 100）：

```
1 • 0 • /sbin/init
    └── 50 • 1 • /usr/bin/launcher
            └── 100 • 50 • /usr/bin/app
                    ├── 200 • 100 • /usr/bin/worker      ← 红色高亮
                    │       └── 201 • 200 • /usr/bin/task
                    └── 300 • 100 • /usr/bin/worker2     ← 红色高亮
                            └── 301 • 300 • /usr/bin/task2
```

多 PID 无共同祖先（LCA = PID 1）时，各组独立输出，空行分隔：

```
1 • 0 • /sbin/init
    └── 50 • 1 • /usr/bin/launcher
            └── 200 • 50 • /usr/bin/worker              ← 红色高亮
                    └── 201 • 200 • /usr/bin/task

1 • 0 • /sbin/init
    └── 60 • 1 • /usr/bin/other
            └── 400 • 60 • /usr/bin/service             ← 红色高亮
```

## 实现：方案 A（进程池 + 标记法）

**完全重写现有脚本**，统一使用进程池方案处理单 PID 和多 PID 场景。优势：
- 只调用一次 `ps`，性能最优
- 合并逻辑在内存中操作，清晰高效
- 单 PID 场景也受益于进程池（不再逐进程调用 `ps`）

## 兼容性

保持对 macOS 和 Linux 的兼容（当前脚本已有的 `uname` 检测逻辑）。
